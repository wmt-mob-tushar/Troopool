R&D Research

The full research blueprint, matching & verification.

R&D and product blueprint for a corporate carpooling app modelled on sRide (https://sride.co/).
Companion file: prototype.html — an interactive, phone-framed mobile flow (open in any browser).

Confidence tags: [C] confirmed from sRide/store listings · [K] competitor-confirmed · [I] inference / industry-standard.

1. The concept in one line

A peer-to-peer office-commute ride-sharing app (not a taxi service) that matches daily commuters on overlapping home→office routes so they share the ride and split the cost — fuel + tolls only, no profit. [C]

sRide model
Primary use Daily, intracity, recurring office commute [C]
Modes Carpool + bikepool + taxipool (split a cab) [C]
Cost Distance + fuel + tolls ÷ confirmed riders → ₹30–₹80/trip; ~70% cheaper than Ola/Uber [C]
Trust anchor Ride only with verified colleagues & professionals [C]
Notable stat ~60% of users are women → safety-first positioning [C]
Commission sRide takes 6% of the contribution [C]

How it differs
- vs Uber/Ola — not on-demand strangers; recurring corridor-matched colleagues, no surge, ~70% cheaper. [C]
- vs BlaBlaCar — BlaBlaCar's core is long-distance one-off trips; its commute product (BlaBlaCar Daily) is a separate app using a fixed transit-"line" model + government subsidies. sRide is daily P2P intracity. [K]

2. Roles — one account, two modes

A single verified profile toggles per trip — a user can be both. [C]

  • Ride Giver (Driver): offers empty seats on an existing commute. Earnings → linked bank account or sRide wallet. Posts a ride in one step; can pick co-riders by their rating. [C]
  • Ride Taker (Rider): books a seat; the per-head cost drops as more riders join (split equally). [C]

UX implication: do not build two account types. Build one account + a prominent Find a Ride / Offer a Ride toggle on the home screen.

3. The full pool flow (step by step)

A · Onboarding & profile — "under 2 min, 4 steps" [C]
1. Mobile number → OTP verify
2. Sign up with work email (corporate verification)
3. Add home + office addresses + work timings
4. Verify profile → start

B · Matching — corridor-level, not city-level [C]
- Finds riders whose routes overlap ≥70%, minimising detour vs driving solo.
- Marketed as a "60-second match."
- Match keys: home→office corridor + departure-time window + recurring day pattern.

C · Giver offers a ride [C] — one-step post: route, available seats, departure time, recurring schedule (weekly with same co-riders, or one-off, flexible day selection) → instant match suggestions.

D · Taker finds a ride [C] — Find a Ride → corridor + preferred time → browse matched riders (rating, route-overlap %, price, gender) → confirm/request seat.

E · Request → accept → confirm [C] — rider requests; driver accepts; rider gets an acceptance notification.

F · During the ride [C] — real-time GPS live tracking (shareable trip status), in-app chat, one-tap SOS.

G · After the ride [C]
- Settlement: auto-deducted on completion (no manual transfer). UPI / card / net-banking / wallet. Cash not supported.
- Wallet / points: earnings → wallet/bank; rewards = badges, points, priority parking, 10% referral credits.
- Ratings: compulsory two-way rating + review; visible before booking.

Splash → OTP → Work-email verify → Gov-ID/KYC → Home addr → Office+timings → (Giver: vehicle)
   → HOME (toggle) ─┬─ Find a Ride → matches → detail → request → confirmed ─┐
                    └─ Offer a Ride → schedule → requests → accept ──────────┤
   → Active ride (map + SOS + chat) → Cost summary (auto-pay) → Two-way rating → Wallet/Points

4. Organization / corporate login (the B2B core)

Employee verification [C]
- Verify in ~2 min with work email.
- No corporate domain (freelancer/generic email) → verify via government ID (Aadhaar / PAN / DL) + workplace address.
- Result: colleague-only / company-scoped corridor circles — "ride only with verified colleagues."

Enterprise onboarding (≈5–7 business days) [C]
1. Leadership alignment (ESG + employee experience + facility-cost story)
2. Employee enrollment via work email
3. Route/corridor mapping — cluster employees by area + timing
4. Pilot (one campus/zone) → scale
5. Incentives: reserved parking, recognition, allowances, leaderboards
6. Ongoing comms (savings milestones)

HR / admin dashboard metrics [C] — adoption rate, active poolers, rides/week, cars removed from roads, CO₂ avoided (ESG / BRSR Scope-3 reportable), aggregated savings, satisfaction. Reference rollouts: Infosys (~800 emp.), Novartis, Capgemini.

Build note: sRide markets "free to companies / zero per-seat licensing" — likely sales positioning. A real B2B line monetises via the HR platform + ESG-reporting tier. [I]

  • Gender preference [C]: women-only ride filter (matches 60% female base).

  • Ratings [C]: compulsory two-way, visible pre-booking.

8. Competitor & build insight

Quick Ride [K] (main rival) — ~3.5M users, points-as-currency, more customizable/feature-rich (group chat, location-share, vacation settings, multi-vehicle, company-domain filter). sRide wins on safety/verification + women-friendliness.

Carpool matching blueprint (Mobisoft, 2026) [I] — directly reusable:
- 6-dimension match score: schedule 30–35% · route geometry 25–30% (PostGIS + Maps detour) · preferences 15–20% (gender = hard filter) · trust/history 10–15% · workplace proximity 5–10% · reliability 5–10%.
- Two-stage matching: coarse geo/time pre-filter (drops ~95% of pairs) → precise Maps detour scoring; daily batch + same-day incremental.
- Suggested stack: React Native app · React+TS admin · Node/NestJS API · Python FastAPI matching service · Socket.io+Redis for GPS · PostgreSQL 16 + PostGIS + TimescaleDB.
- UX wins: calendar integration (auto-detect office days), offline-first driver app, pre-ride message templates, same-department social context, trip-level cost+CO₂ transparency, multi-channel SOS.

9. Automated document verification — DigiLocker & KYC aggregators

Goal: verify Aadhaar (identity), Driving Licence and Vehicle RC (driver-eligibility gate), and PAN — fully automated, zero manual review. This is reflected in the prototype's KYC screen.

9.1 Is there an official DigiLocker API? — Yes [C]

DigiLocker (MeitY / NeGD) exposes a consent-based Partner / Requester API (OAuth 2.0 via Meri Pehchaan). Your app redirects the user to DigiLocker, they log in and consent, and you pull issuer-signed documents back — verified at source, not user-uploaded files.
- What you can pull (issuer-verified): Aadhaar (e-KYC XML), Driving Licence (from Sarathi/MoRTH), Vehicle RC (from VAHAN/MoRTH), PAN (CBDT), education & other issued docs. Data is signed by the issuer → authentic, tamper-evident.
- Eligibility / onboarding: the direct DigiLocker partner programme (partners.digitallocker.gov.in, apisetu.gov.in) is geared to govt/regulated/large entities, has an approval process + sandbox→production, and is slow & heavy for a small startup. [C/I]

9.2 Aadhaar reality in 2026 [C — material]

  • Standalone Aadhaar OTP-OKYC has been largely discontinued across providers after UIDAI technical changes (e.g. Setu retired its OKYC product). → DigiLocker is now the de-facto industry route to Aadhaar data, not a co-equal alternative.
  • Aadhaar paperless offline e-KYC (XML/ZIP, user-generated, UIDAI-signed) still works and is offered by AuthBridge, Perfios/Karza, Digitap. So your real Aadhaar paths are DigiLocker or offline XML — you do not need to become an AUA/KUA.

9.3 The practical startup path — aggregators that wrap DigiLocker [C/K]

Because direct partnership is hard, use an aggregator that fronts DigiLocker + VAHAN(RC) + Sarathi(DL) + PAN behind one REST API/SDK.

Provider Covers Self-serve? Notes
Setu DigiLocker, PAN ✅ sandbox + free credits Only vendor with a public price: ₹4 / successful DigiLocker fetch. Cleanest docs. Narrow — no standalone DL/RC, OKYC dead.
Surepass DigiLocker, DL, RC, PAN, offline Aadhaar ✅ Richest public SDKs (iOS/Android/React Native/Web). Fullest coverage + self-serve.
Perfios / Karza (Perfios Hub) DigiLocker, DL, RC, PAN, offline Aadhaar ✅ Fullest coverage + sandbox/free credits. Enterprise-grade.
Cashfree Verification DigiLocker, RC, PAN, bank/UPI ✅ Public Python verification SDK on GitHub. Good if already using Cashfree payments.
Zoop.one / Digitap DigiLocker, DL, RC, PAN, offline Aadhaar ✅ Credits visible post-signup (no public rate card).

Bureau — — Wrong tool: fraud/risk-decisioning, no public India doc-level KYC endpoints.

"Transparent pricing" for Surepass/Zoop/Digitap means free trial + credits after signup, not a public rate card. Only Setu publishes an INR per-call rate. [C]

9.4 DL & RC specifically (the driver gate)

Beyond DigiLocker-issued copies, aggregators also expose live Sarathi (DL) and VAHAN (RC) lookups — better for "verify the driver before they can offer a ride," since you confirm the licence/registration is currently valid. Surepass, Cashfree, Perfios/Karza do these standalone; Setu does not. [C]

9.5 Recommended architecture (fully automated, startup-friendly) [I, grounded in 9.1–9.4]

  1. Identity: DigiLocker consent flow → issuer-signed Aadhaar e-KYC + PAN (also proves document ownership). Vendor: Setu (clean, ₹4/fetch) or Surepass/Cashfree if you want one vendor for everything.
  2. Driver gate: live DL (Sarathi) + RC (VAHAN) checks via Surepass / Cashfree / Perfios before "Offer a ride" unlocks.
  3. Identity binding: DigiLocker consent (strongest) or selfie-liveness + face-match to the DL photo (add HyperVerge/IDfy SDK for best-in-class liveness).
  4. Fastest route: a single self-serve aggregator covering all five — Surepass or Perfios Hub — gets you a fully automated gate today; add Setu if you specifically want the transparent-priced clean DigiLocker fetch.
  5. Compliance: explicit consent capture + audit log; DPDP Act 2023 — collect minimum necessary, define retention, prefer storing verification result + reference ID over raw KYC payloads; encrypt at rest; honour erasure requests.

Bottom line: Yes — DigiLocker is exactly the right primitive, and you avoid manual review entirely. Don't chase direct DigiLocker partnership first; ship on Surepass or Perfios Hub (DigiLocker + DL + RC + PAN + offline Aadhaar, self-serve), optionally add Setu for clean priced DigiLocker fetches and HyperVerge/IDfy for liveness.

10. Caveats before mirroring sRide 1:1

  • sRide's own stats conflict across pages (users 1–3M, cities 9–20+, companies 300–50,000) — pick one internally consistent set.
  • "Free to companies" is likely positioning — plan a real B2B revenue line.
  • RC/DL-for-givers verification is industry-standard but not explicitly stated by sRide — confirm in-app.
  • 70% overlap and "60-second match" are sRide marketing figures — fine to clone as UX copy.
  • KYC vendor prices (except Setu's ₹4/DigiLocker fetch) are signup-gated — get quotes before committing; Signzy's annual minimum estimate is third-party, unverified.

11b. sRide FAQ — confirmed flow (verbatim from sride.co/faqs) [C]

Pulled directly from the FAQ; the prototype mirrors all of this:
- Sign-up (<2 min): download → mobile + OTP → add home & office → complete verification → start matching.
- 3-layer verification: Mobile OTP · work email / company ID · Government ID + selfie match (Aadhaar/PAN/DL). Freelancers verify via ID + workplace address.
- One account, both roles: "switch between Driver and Rider at any time."
- Matching: routes overlap ≥70%, detours minimised. No match → added to "corridor waitlist", notified when a matching route joins. ← add to prototype
- Booking: Find a Ride → select corridor + time → browse matches → confirm seat.
- Recurring: "weekly recurring rides with the same co-riders, or one-off rides." ✅ built.
- Fare: distance + fuel + tolls ÷ confirmed riders; auto-deducted after ride; UPI/card/netbanking/wallet; no cash. Saves ₹2,500–6,000/mo.
- Safety: women-only preference · live GPS shared with contacts · in-app SOS · "Report Ride" post-trip; Trust & Safety reviews within 24h. ← add Report Ride
- Cancellation: instant notification + auto-attempt to find an alternate match; frequent cancellers restricted.
- Corporate: enterprise solution at 200+ companies — dashboards, bulk onboarding, analytics, subsidised rides; 20–40% parking reduction in 3 months.

12. Similar applications (competitor landscape) [K/I]

India — direct peers (office carpooling):
- Quick Ride — sRide's main rival; ~3.5M users, points-as-currency (1pt=₹1, ~4pts/km), 6% fee, more configurable (group chat, vacation mode, multi-vehicle, company-domain filter). Strong Bengaluru/Chennai/Pune.
- Orahi — office/community carpooling; higher commission historically (~28%).
- Zify — dynamic carpooling on your route, intercity + intracity; large SE-Asia/India footprint.
- MoveInSync / Commute (enterprise) — employee-transport SaaS (managed cabs + carpool); strong corporate/HR angle.
- Ola/Uber — on-demand cabs (not cost-sharing); the price benchmark sRide undercuts ~70%.

Global — for feature inspiration:
- Scoop (US) — "request the night before, paired for the morning"; as-needed corporate carpool, employer-funded. Great recurring/scheduling UX.
- Pave Commute — work carpool + rewards for sustainable commuting; ESG-led.
- Comovee — maps nearby colleagues geographically to form company carpools.
- Commute with Enterprise — vanpool (provided SUV/van, riders rotate driving).
- ShareSpace Carpool — co-worker ride-sharing, "save up to $300/mo."
- BlaBlaCar / BlaBlaCar Daily — long-distance marketplace; the Daily app uses a fixed transit-"line" + subsidies for commutes.
- Hitch — city-to-city long-distance rideshare.
- Waze Carpool — (discontinued 2022) Google's commute-matching experiment; useful cautionary case.

Takeaway for Troopool: sRide/Quick Ride/Orahi/Zify own the Indian P2P-cost-sharing model; Scoop/Pave/Comovee show the employer-funded + scheduling UX worth borrowing. Differentiate on verification/safety + women-friendliness + transparent CO₂/ESG, which is exactly sRide's wedge.
