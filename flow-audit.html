Flow Audit & Finalized Flow

What was weak, and the finalized end-to-end product flow.

Senior flow-master review of the v1 prototype (troopool-app.html, ~37 screens, 5 tabs) → the finalized flow that troopool-v3.html implements.

A. Current-state map (v1)

Tabs: Discover · Commute · Circles · Chats · You.
Carpool: find→matches→detail→request→confirmed→active→cost→rating · offer→requests→accept/decline · recurring · taxi→taximatch→taxiconfirm.
Community: deck→person→connect→match→chat (carpool card) · circle→circleChat · circleMatch · planNew→planCreated · neighbor.
State globals: HOME_MODE, DRIVER_VERIFIED, BOOKING, CARPOOL, CIRCLE_CARPOOL, CTX, DTAB.

B. Weaknesses (prioritized)

P0 — architecture / clarity

  1. Discover ↔ Commute redundancy. Both let you start a carpool via different mental models → users don't know which to use.
    Fix: Commute = the only ride-booking hub; Discover = social exploration. From a Discover person you propose a carpool (into chat), you don't book there.
  2. Community→carpool bridge is asymmetric. "Pool together" only shows for route matches (p.why.k==='way'); interest/neighbour connections can't pool.
    Fix: any connection can "Propose a carpool" via a sheet (route/time) that drops a carpool card into the chat.

P1 — high

  1. Taxipool dead-ends — taxiconfirm has no CTA; taxipools never reach My Rides. Fix: add CTAs (chat / meeting-point / track) + log to history.
  2. Chat carpool card is manual — needs an explicit tap to appear. Fix: auto-surface for route-matched people.
  3. State fragility / double-booking — CARPOOL and CIRCLE_CARPOOL bypass the one-per-slot BOOKING gate. Fix: unify into one BOOKING{type, who, slot, status} with a conflict sheet.

P2 — medium

  1. Onboarding doesn't branch rider vs host (KYC friction discovered at point-of-use). Fix: ask role after email-verify; riders skip KYC.
  2. Plans half-baked — no "My Plans/Going/Invited", no edit/cancel. Fix: add a My Plans view.
  3. circleMatch is read-only — no "Say hi" interaction. Fix: Say-hi → mutual → group chat opens.
  4. Recurring has only skip/pause. Fix: add edit time/route + cancel.

P3 — low

  1. My Rides rider/driver toggle has no visible control. Fix: add a segmented toggle.
  2. Wallet & sNeighbour are near-orphans. Fix: link Wallet from You; flesh out sNeighbour (request→accept→chat→done).

Genuinely SOLID (kept as-is in v3)

Rider booking chain (one-per-slot BOOKING) · driver path (KYC→vehicle→offer→requests) · recurring skip/pause · route map + transparent cost (match-ring %) · person→connect→match · circle detail (members/plans/group-chat/carpool-in-circle) · circles join/leave.

C. Finalized to-be flow (v3)

IA — 5 tabs, Discover is the landing (community-first)

  • Discover — grouped feed (🛣️ On your way / 🏠 Nearby / 🎸 Wavelength) + swipe deck + "Happening near you" → person → connect → match. Carpool here = Propose a carpool (→ chat), not booking.
  • Commute — the transactional ride hub: Find · Host · Recurring · Taxipool · My Rides. Cross-links to Discover.
  • Circles — your circles + suggested + Meet other circles (group-to-group Say-hi→mutual→chat) → detail (members · plans · carpool-in-circle · group chat).
  • Chats — list → chat that auto-shows a carpool card when route-matched; action chips (Carpool / Coffee / Walk / Plan).
  • You — profile · verification · interests/open-to · My Rides · Wallet · Plans · settings (women-only, SOS) · HR/ESG dashboard (corporate).

Unified state

BOOKING = {type:'carpool'|'circle'|'taxi', who, slot, status} — one active ride per slot, across all types; conflict sheet on double-book. DRIVER_VERIFIED gate; CTX{person,circle}.

Happy paths (the demo)

  • B — Community (primary): onboard → Discover → person(why-overlap) → connect → match → chat(chips) → join circle → RSVP/My Plans → sNeighbour.
  • A — Carpool (wedge): Discover "on your way" person → Propose carpool → chat carpool card → Confirm → active(map/SOS) → cost → rating → next-day recurring. (Or Commute → Find → matches → detail → request → confirmed.)
  • C — Corporate (B2B): You → HR/ESG dashboard (adoption · active poolers · cars removed · CO₂ Scope-3 · ₹ savings · corridors).
  • D — sNeighbour: feed → accept airport-pool / lend item → chat → done.

Swimlane (Journey A, text BPMN)

Rider          | open Discover → tap person → Propose carpool → chat → Confirm → [ride] → rate
System         |  match% + why-overlap → create BOOKING(proposed) → carpool card → BOOKING(confirmed) → auto-pay → 2-way rating → seed recurring
Driver/Host    |                                   ← accept request (Commute path) ←            → rate rider

D. What changes vs v1

Disambiguate Discover/Commute · unify carpool state · universal Propose-carpool bridge · fix taxipool/chat-card/plans/recurring/myrides/wallet/sNeighbour · onboarding role branch · new theme (indigo-led "Connect" palette) · full scope incl. HR/ESG dashboard. Build = clean troopool-v3.html from scratch, v1+v2 as reference.
